# SAMPLE SOURCING & LIBRARY MANAGEMENT KNOWLEDGE BASE
## For DAW Brain — Finding, Organizing & Managing Production Assets

**Practical reference for sourcing quality samples, managing file formats, organizing libraries, DJ library integration, and backup strategy.** Tailored for groovy underground tech house at 124–130 BPM on a 2020 Intel MacBook Pro with 8GB RAM, using Ableton Live 12 Suite.

Reference artists: YOBBO, CHASEWEST, BELTRAN, SLAMM, CHALANT, Jamie Jones, The Martinez Brothers, Patrick Topping, Solardo

## 6.1 SPLICE STRATEGY

Splice is the primary source for production-ready one-shots and loops.

### Credit Management

- **Plans:** Sounds+ ($12.99/month, base credits), Creator ($19.99/month, 200 credits + AI tools), Creator+ (500 credits). Creator Annual ~$14/month (30% savings). Credits roll over while subscription is active.
- **Credit costs:** All samples = 1 credit. MIDI patterns and presets = up to 3 credits.
- **Serum 2 Rent-to-Own:** ~$9.99/month for 25 months, no interest, cancel anytime. If already renting Serum 1, you get Serum 2 access automatically. Presets are free to download while renting.
- **If you cancel:** Unused credits freeze and expire after 28 days. You keep everything already downloaded permanently.
- **Strategy:** Spend 80% of credits on one-shots (kicks, claps, hats, percussion, vocal chops). Spend 20% on loops (top loops, percussion loops, FX risers). One-shots are more versatile — a single great kick appears in dozens of tracks.
- **Don't hoard credits.** If you have 300+ credits banked, you are not using the service. Dedicate 20 minutes per week to browsing and downloading with intention.

### Search Terms for Tech House

Splice search is keyword-based. These exact terms return the best results for underground tech house:

**Kicks:**
- `tech house kick punchy` — tight, mid-range focused kicks
- `909 kick processed` — classic analog kick character
- `minimal kick` — clean, less processed starting points
- `kick sub layer` — dedicated sub kicks for layering

**Claps & Snares:**
- `tech house clap dry` — tight, no reverb
- `clap snare layer` — pre-layered clap+snare hits
- `909 clap` — classic analog clap
- `snare rim` — tighter snare variants

**Hi-Hats:**
- `closed hat tight` or `hi hat closed crisp` — main rhythm hats
- `open hat 909` — classic open hats for offbeat groove
- `hat shuffle` — pre-swung hat patterns (use as reference, reprogram your own)

**Percussion:**
- `conga` `bongo` `rim shot` — Latin percussion for Martinez Brothers-style layers
- `shaker 16th` — steady shaker patterns
- `percussion tribal` — organic percussion elements
- `ride cymbal` — subtle ride patterns for momentum

**Bass:**
- `bass one shot tech house` — single bass hits for Simpler/Sampler
- `bass loop 128` — pre-made bass patterns (filter: 124–130 BPM)
- `sub bass one shot` — clean sine sub hits

**Vocals:**
- `vocal chop tech house` — pre-chopped vocal snippets
- `vocal phrase female` or `vocal phrase male` — short sung phrases
- `vocal ad lib` — individual vocal ad-libs (hey, yeah, uh)
- `vocal processed` — already filtered/effected vocals

**FX:**
- `riser white noise` — build tension
- `downlifter` — post-drop transitions
- `impact sub` — beat 1 drop hits
- `transition fx tech` — assorted transition elements
- `reverse cymbal` — classic pre-section FX

### Filtering

Always filter results by:
- **BPM:** 124–130 (set this first to avoid tempo mismatches)
- **Key:** Match your project key, or filter by compatible keys (same Camelot number ±1)
- **Genre:** Tech House, Minimal, Deep Tech (the genre tags overlap — check all three)
- **Instrument:** Use this to narrow drum types (kick, snare, hat) or instrument categories

### Preview Workflow

1. Open Splice desktop app alongside your Ableton project
2. Set your Ableton project tempo (e.g., 128 BPM)
3. In Splice, samples auto-sync to your DAW's tempo when using the bridge feature
4. Preview samples against your playing project — listen for groove compatibility, not just sound quality
5. Download only samples that survive the "does this fit the groove?" test — not samples that sound cool in isolation

### Favorite Packs for Tech House

Search by these artists/labels on Splice:
- **Toolroom Records** — consistently high-quality tech house sample packs
- **Cr2 Records** — deep tech and tech house focused
- **ARTFCL** — underground tech house percussion and drums
- **Sample Magic** — wide range, good FX and vocal packs
- **Oliver Heldens / Heldeep** — groovy house drums and bass
- **Roger Sanchez** — classic house drums, soulful elements
- **Splice Originals: Tech House** — curated by Splice editors, consistently good

### One-Shots vs. Loops

**One-shots (preferred 80% of the time):**
- Single drum hits, bass notes, vocal chops, FX hits
- You control the rhythm, timing, velocity, and processing
- More versatile — one kick works in 50 tracks
- Load into Simpler or Drum Rack for playability
- Lower CPU than streaming audio loops

**Loops (20% of the time):**
- Top loops (percussion patterns) for layering over programmed drums
- Background texture loops for breakdowns
- Reference loops to study groove patterns (then reprogram your own)
- Use in Session View for auditioning, then commit the loop or reprogram it

### Serum Presets on Splice

Splice's Rent-to-Own program for Serum 2 includes access to thousands of community presets:
- Browse by category: Bass, Lead, Pad, Pluck, FX
- Filter by genre: House, Tech House, Minimal
- Free to download while renting Serum (no credit cost)
- **Quality varies wildly.** Preview before downloading. Many presets are CPU-heavy with excessive unison voices and effects — strip these back for 8GB RAM systems.
- Best use: download as starting points, then modify. Never use a preset unmodified in a release.

## 6.2 OTHER SAMPLE SOURCES

### Qobuz (Lossless Full Tracks)

Qobuz is the primary source for lossless full tracks used as sampling source material and for the DJ library.

- **Subscription:** Sublime ($14.99/month) for unlimited streaming + discounted purchases. Studio quality: up to 24-bit/192kHz FLAC.
- **Purchases:** Buy individual tracks or albums in FLAC or WAV. Purchased tracks are yours permanently — no DRM.
- **Why Qobuz over others:** Lossless quality. Spotify/Apple Music are lossy (AAC/OGG). Tidal offers lossless but Qobuz's download-to-own model is better for sampling (you own the file, no DRM restrictions on loading into a DAW).
- **Workflow:** Search for source material (hip-hop, soul, disco records for sampling). Purchase in FLAC. Convert to WAV if needed (see 6.3). Load into Ableton for chopping.
- **For DJ library:** Purchase tracks in FLAC, convert to 16-bit/48kHz WAV for Rekordbox/CDJ compatibility (see 6.3 and 6.5).

### Beatport / Beatport Sounds

- **Beatport** (main site): DJ tracks for purchase. WAV format available. Use for DJ library and reference tracks. Filter by genre (Tech House), key, BPM.
- **Beatport Sounds:** Sample packs. Higher price point than Splice ($15–$30 per pack) but curated for electronic genres. Good for genre-specific drum kits and producer signature packs.
- **Advantage:** Beatport tags tracks with verified key and BPM — useful for confirming sample key before production.

### Loopmasters

- Large catalog of sample packs, skewed toward electronic/dance music.
- Packs are more expensive ($20–$40) but contain complete production toolkits: drums, bass, synths, FX, MIDI files.
- **Best for:** When you want a cohesive pack where all elements are designed to work together.
- Filter by genre: Tech House, Deep Tech, Minimal House.
- Download includes WAV files + Rex2 + Ableton Live Clips for some packs.

### Scraps Audio (Artist Templates)

Ableton project templates in the style of reference artists. Excellent for studying arrangement, mixing, and processing decisions.
- **CHASEWEST / Solid Grooves — Tech House:** Stock Ableton plugins only, requires Live 11.3.10+. Proves the stock-only production approach.
- **BELTRAN / Solid Grooves — Tech House Style:** Stock plugins, Live 11.3.10+. Multiple templates available.
- **Tech House Serum Presets:** Serum preset pack for tech house bass, leads, pads.
- Use case: overcome blank-page syndrome, study arrangement structure, learn mixing techniques from deconstructed references.

### Tech House Market

Dedicated marketplace for tech house production resources. Includes BELTRAN-specific project files and stems. Free downloads section available.

### Free Sources

**Ableton's Built-In Library (best free source):**
- Extensive drum hits, synth one-shots, loops, and MIDI clips included with Live 12 Suite
- Core Library + Curated Collections cover most basic needs
- **Underused resource:** The 909, 808, and 707 kits in the Drum Machines folder are production-ready for tech house
- Access via Browser → Packs → Core Library → Samples

**Cymatics:**
- Free sample packs (sign up required, email marketing heavy)
- Quality is mixed — download the free packs, cherry-pick the best hits, delete the rest
- Good for: FX one-shots, vocal chops, basic drum hits
- Not reliable enough to be a primary source

**LANDR Samples:**
- Free tier gives limited monthly downloads
- Quality is moderate. Better than Cymatics, not as curated as Splice.
- Good as a supplementary source, not primary.

### Recording Your Own

**Field recordings:**
- Use your iPhone voice memo app for quick captures (ambient sounds, found percussion, crowd noise)
- Import into Ableton, process heavily: HP filter at 200 Hz, compression, saturation, time-stretch
- A recorded door slam processed through a pitch envelope becomes a unique kick layer
- A recorded glass clink becomes a unique percussion hit
- Unique sounds that no other producer has — the ultimate differentiation

**Vocal recordings:**
- Record yourself or collaborators with any USB microphone into Ableton
- Even low-quality vocal recordings become usable after: pitch correction (if needed), heavy filtering, chopping, and processing
- Short phrases (1–3 words) work best for tech house — process into rhythmic chops

### YouTube (Legal Considerations)

- **Quality:** YouTube streams at lossy AAC, typically 128–256 kbps. This is below production quality. Artifacts compound through processing.
- **Legal:** Downloading and sampling YouTube audio violates YouTube's ToS and likely infringes copyright. Do NOT sample directly from YouTube for released tracks.
- **Acceptable use:** YouTube is fine for reference listening, studying arrangements, and finding source material — then purchase the actual track on Qobuz/Beatport in lossless format for sampling.

## 6.3 FILE FORMAT BEST PRACTICES

### Format Hierarchy

| Format | Type | Use Case | Quality |
|---|---|---|---|
| **WAV 24-bit/44.1kHz** | Uncompressed | Production (default) | Lossless, maximum headroom |
| **WAV 16-bit/44.1kHz** | Uncompressed | Final exports, CD-quality | Lossless, standard quality |
| **WAV 16-bit/48kHz** | Uncompressed | DJ files for CDJs | Lossless, CDJ-optimized |
| **FLAC** | Lossless compressed | Archival, storage | Lossless, ~60% of WAV size |
| **AIFF** | Uncompressed | Alternative to WAV (Mac) | Lossless, same quality as WAV |
| **MP3 320kbps** | Lossy | Demos, streaming, mobile | Near-transparent but lossy |
| **AAC/M4A** | Lossy | Apple ecosystem | Similar to MP3, not for production |

### Rules

1. **Production:** Always work in WAV 24-bit/44.1kHz. The extra 8 bits of headroom (over 16-bit) matter when stacking processing chains. 44.1kHz is the standard — 48kHz offers no audible benefit for music production and uses more disk space.
2. **Final export from Ableton:** WAV 24-bit/44.1kHz for mastering. WAV 16-bit/44.1kHz for distribution (DistroKid, LANDR) and DJ use on CDJs (safest universal format).
3. **Archival:** Convert WAV to FLAC for long-term storage. FLAC is ~40% smaller with zero quality loss. **Important: Ableton does NOT import FLAC directly** — convert back to WAV before loading into a project.
4. **Never convert lossy → lossy.** MP3 → M4A or M4A → MP3 compounds compression artifacts. Each generation of lossy compression removes different frequency information. After two conversions, the audio is noticeably degraded.
5. **Never upsample lossy to lossless.** Converting an MP3 to WAV does NOT restore the lost quality. The WAV file is just a larger container holding the same degraded audio. If a sample was downloaded as MP3, it stays MP3-quality forever.

### CDJ Compatibility (Pioneer/Rekordbox)

Pioneer CDJs (2000NXS2, 3000) support:
- **WAV:** 16-bit and 24-bit, 44.1kHz and 48kHz. **Safest format: 16-bit/44.1kHz WAV** — works on every Pioneer CDJ ever made. No compatibility surprises.
- **FLAC:** Supported on CDJ-3000 and newer. NOT supported on older CDJ-2000NXS2. Risky for gigs at venues with older gear.
- **AIFF:** Fully supported on all CDJ models. Good alternative to WAV (supports metadata/artwork natively).
- **MP3:** Supported but not recommended — lossy quality is audible on club systems.
- **AAC/M4A:** Supported on newer models. Not recommended for professional use.

**WAV_EXTENSIBLE Bug (Critical):**
Some WAV files (including hi-res downloads from Bandcamp and some converters) contain a hidden header property called `WAV_EXTENSIBLE` that Rekordbox ignores but CDJs reject with error E-8305 ("Unsupported File Format"). The track plays fine in Rekordbox but fails on the actual CDJ at the gig.
- **Fix:** Re-encode the file with FFmpeg (even to the same specs): `ffmpeg -i broken.wav -c:a pcm_s16le -ar 44100 fixed.wav`
- **Prevention:** Always re-encode downloaded WAVs through FFmpeg before adding to your DJ library. The batch command in section 6.3 handles this automatically.

### FFmpeg Conversion Commands

FFmpeg is the standard command-line tool for audio conversion. Install via Homebrew: `brew install ffmpeg`

**FLAC → WAV (lossless, for production):**
```bash
ffmpeg -i input.flac -acodec pcm_s24le -ar 44100 output.wav
```
- `pcm_s24le` = 24-bit WAV. Use `pcm_s16le` for 16-bit.
- `-ar 44100` = 44.1kHz sample rate. Use `48000` for DJ files.

**WAV → FLAC (lossless compression, for archival):**
```bash
ffmpeg -i input.wav -acodec flac -compression_level 8 output.flac
```
- `compression_level 8` = maximum compression (smallest file, slower encode). Level 5 is a good balance.

**WAV → WAV (resample for CDJs: 24-bit/44.1kHz → 16-bit/48kHz):**
```bash
ffmpeg -i input.wav -acodec pcm_s16le -ar 48000 output_dj.wav
```

**Batch convert all FLACs in a folder to 24-bit WAV:**
```bash
for f in *.flac; do ffmpeg -i "$f" -acodec pcm_s24le -ar 44100 "${f%.flac}.wav"; done
```

**Batch convert all WAVs to DJ-ready 16-bit/44.1kHz (also fixes WAV_EXTENSIBLE bug):**
```bash
for f in *.wav; do ffmpeg -i "$f" -acodec pcm_s16le -ar 44100 "dj_${f}"; done
```

### Sample Rate: 44.1kHz vs 48kHz

- **44.1kHz:** The CD standard. All streaming platforms (Spotify, Apple Music, Beatport) distribute at 44.1kHz. Ableton's default project sample rate. Use this for production.
- **48kHz:** The video/broadcast standard. Also the internal processing rate of modern Pioneer CDJs. Use this for DJ files only.
- **Mixing sample rates:** If your Ableton project is 44.1kHz and you import a 48kHz sample, Ableton automatically resamples it. This is transparent — no audible quality loss. But it costs a tiny amount of CPU. Keep all production samples at 44.1kHz for consistency and efficiency.

## 6.4 LIBRARY ORGANIZATION

### Folder Structure

Organize your sample library by function, not by source. A kick from Splice and a kick from Loopmasters should live in the same folder.

```
Samples/
├── Drums/
│   ├── Kicks/
│   ├── Claps-Snares/
│   ├── Hi-Hats/
│   ├── Percussion/
│   │   ├── Congas-Bongos/
│   │   ├── Shakers/
│   │   ├── Rimshots/
│   │   └── Other/
│   └── Loops/
│       ├── Top-Loops/
│       └── Full-Drum-Loops/
├── Bass/
│   ├── One-Shots/
│   └── Loops/
├── Synths/
│   ├── Stabs/
│   ├── Pads/
│   ├── Leads/
│   └── Plucks/
├── Vocals/
│   ├── Chops/
│   ├── Phrases/
│   ├── Ad-Libs/
│   └── Processed/
├── FX/
│   ├── Risers/
│   ├── Downlifters/
│   ├── Impacts/
│   ├── Textures/
│   └── Transitions/
├── Loops/
│   ├── Melodic/
│   └── Textural/
└── Source-Material/
    ├── Full-Tracks/
    └── Stems/
```

### Naming Conventions

Rename samples as you organize. A descriptive filename saves hours of browsing later.

**Pattern:** `Type_Character_BPM_Key.wav`

**Examples:**
- `Kick_Punchy_128_E.wav`
- `Clap_Dry_Tight_128.wav`
- `HiHat_Closed_Crisp.wav`
- `Conga_High_Latin.wav`
- `Bass_Bouncy_128_Fm.wav`
- `Vocal_Chop_Female_Hey.wav`
- `Riser_WhiteNoise_8bar.wav`
- `Impact_Sub_Drop.wav`

**Rules:**
- No spaces in filenames — use underscores. Spaces cause issues with some command-line tools and older software.
- BPM only for tempo-dependent samples (loops, bass loops). Omit for one-shots that work at any tempo.
- Key only for tonal samples (bass, synths, vocals, melodic loops). Omit for atonal samples (kicks, claps, hats, noise FX).
- Keep names under 50 characters. Ableton's browser truncates long names.

### Ableton Browser Integration

**Places:**
1. Open Ableton's Browser (Cmd+Opt+B)
2. In the sidebar, right-click under "Places"
3. Click "Add Folder..."
4. Navigate to your `Samples/` root folder and add it
5. Your entire organized library now appears in Ableton's browser with search, preview, and drag-and-drop
6. Add multiple Places for different drives (internal SSD, external sample drive)
7. **Performance warning:** Only add specific, relevant folders — NOT entire drives. Large folder trees cause slow indexing and browser lag on 8GB RAM systems. Add `Samples/Drums/` and `Samples/Bass/` separately rather than adding the root `Samples/` folder.

**Collections (Color-Coded Quick Access):**
1. Right-click any sample, folder, preset, or plugin in the browser
2. Assign it to a Collection (color-coded categories: 1–7)
3. Recommended Collection assignments:
   - **Red (1):** Favorite kicks (the 10 kicks you use most)
   - **Orange (2):** Favorite claps/snares
   - **Yellow (3):** Favorite hats and percussion
   - **Green (4):** Go-to bass one-shots and patches
   - **Cyan (5):** Vocal chops and phrases
   - **Blue (6):** FX (risers, impacts, transitions)
   - **Purple (7):** Full project templates and rack presets
4. Access Collections instantly from the browser sidebar — no folder navigation needed

**Tags (Live 12):**
- Live 12 added user-definable tags to any browser item
- Tag samples with descriptors: `dark`, `bright`, `punchy`, `heavy`, `light`, `filtered`, `latin`, `909`, `acoustic`
- Search by tag in the browser search bar using `#tagname`
- More granular than Collections, complementary to folder organization
- Tag as you go — add tags when you first use a sample in a project

**Saved Searches:**
- Click the **+** button in the Results header to save a browser view including search keywords and active filters
- Creates a custom label in the browser sidebar under Library
- Use for recurring searches: "tech house kick one-shot," "vocal chop processed," "riser white noise"

### Regular Cleanup

Schedule a monthly 30-minute cleanup session:
1. Delete samples you downloaded but never used (Splice makes this easy — re-download if needed)
2. Remove duplicate samples across folders
3. Ensure new downloads are properly renamed and filed (don't let the Downloads folder accumulate)
4. Audit Collections: remove items that no longer deserve "favorite" status
5. Target: keep your active library under 10GB. A smaller, curated library is faster to browse and easier to back up than 100GB of hoarded samples.

## 6.5 REKORDBOX LIBRARY MANAGEMENT

For producers who also DJ, maintaining a parallel DJ library in Rekordbox requires a consistent workflow.

### Importing Production Tracks into DJ Library

1. Export your finished track from Ableton: WAV 16-bit/48kHz (CDJ standard)
2. Save to a dedicated folder: `DJ Library/Own Productions/`
3. In Rekordbox, drag the file into your collection
4. Rekordbox auto-analyzes: BPM, key, waveform, beat grid
5. **Verify the analysis:** Check that Rekordbox detected the correct BPM and key. If wrong, manually correct (right-click → Edit).
6. For edits/bootlegs: add `[EDIT]` or `[BOOTLEG]` to the track title so you know it is not an official release

### Tagging Strategy

Use Rekordbox's built-in metadata fields consistently:

- **Genre:** Tech House, Deep Tech, Minimal, Bass House (match Beatport genre categories)
- **Color:** Use Rekordbox track colors for energy level:
  - **Green:** Low energy (warm-up, intro sets)
  - **Yellow:** Medium energy (building, cruising)
  - **Orange:** High energy (peak time)
  - **Red:** Maximum energy (closer, headline moments)
- **Rating (stars):** Reserve 5 stars for tracks you KNOW work on the dancefloor. 4 stars for strong tracks. 3 stars for "maybe." Below 3 = delete from DJ library.
- **Comments field:** Add specific notes: "big room drop," "vocal hook at 1:30," "mixes well with Fisher," "own production"
- **My Tag:** Rekordbox's custom tag system. Create tags like: `opener`, `peak`, `closer`, `vocal`, `instrumental`, `latin perc`, `dark`, `uplifting`. Enable **"Add My Tag to Comments"** in Preferences → Advanced → My Tag — this makes your tags visible on CDJs in the Comments field.

### Hot Cues & Memory Cues

**Hot Cues (for performance — saved to USB):**
- **A (Red):** First beat of the track (cue point for loading)
- **B (Orange):** Start of the main vocal or hook
- **C (Yellow):** Start of the drop / main groove
- **D (Green):** Breakdown start
- **E (Cyan):** Build/riser start
- **F (Blue):** Outro mix-out point (32 bars from end)

**Memory Cues (for preparation — visual markers):**
- Mark every major section boundary (intro, build, drop, breakdown, drop 2, outro)
- Add comments to memory cues: "kick enters," "bass drops," "vocal starts," "mix out here"
- These appear on the CDJ waveform display as visual guides

### Playlist Organization

```
Playlists/
├── By Energy/
│   ├── Warm-Up (120-126 BPM)
│   ├── Building (126-128 BPM)
│   ├── Peak Time (128-130 BPM)
│   └── Closers (126-130 BPM, vocal/melodic)
├── By Style/
│   ├── Underground Tech House
│   ├── Vocal Tech House
│   ├── Latin/Percussion Heavy
│   └── Dark/Minimal
├── Gig Prep/
│   ├── [Venue Name] [Date]
│   └── ...
└── Own Productions/
    ├── Finished
    ├── DJ Edits
    └── Bootlegs
```

**Gig prep workflow:**
1. Create a new playlist for each gig: `[Venue] [Date]`
2. Pull 40–60 tracks from your categorized playlists
3. Sort by energy level (green → red)
4. Practice transitions between 10–15 key pairings
5. Export to USB: right-click playlist → Export → select USB drive

### Sync Between Production and DJ Libraries

- **One-way flow:** Production → DJ Library. Never the reverse.
- Export finished tracks and edits to `DJ Library/Own Productions/`
- Re-import into Rekordbox when the export changes
- Do NOT store Rekordbox's database on the same drive as production samples — keep them on separate partitions or drives to avoid confusion
- Monthly sync: check if any new productions need adding to the DJ library

## 6.6 BACKUP STRATEGY

Data loss is permanent. A backup strategy is not optional.

### The 3-2-1 Rule

- **3 copies** of every important file
- **2 different storage media** (internal SSD + external SSD, or SSD + cloud)
- **1 copy offsite** (cloud storage or a drive stored elsewhere)

### Storage Layout

**Internal SSD (Boot Drive):**
- macOS and applications
- Current active Ableton projects (the 2–3 projects you are working on right now)
- Ableton's User Library and preferences
- Do NOT store your full sample library here — 8GB RAM machines have limited SSD space

**External SSD (Primary Sample & Project Drive):**
- Full sample library (`Samples/` folder from section 6.4)
- All Ableton project folders (current + archived)
- Rekordbox library and exported USB files
- **Recommended:** Samsung T7 or SanDisk Extreme — USB 3.2, fast enough for real-time audio streaming, 1TB minimum
- Connect via USB-C or Thunderbolt. Do NOT use a spinning HDD — seek times cause audio dropouts.

**Cloud Backup (Offsite Copy):**
- Ableton project files (`.als` files and MIDI/audio within the project folder)
- Serum presets and custom racks
- Use iCloud Drive, Dropbox, or Google Drive — whichever you already pay for
- **Do NOT sync sample libraries to cloud** — they are too large (10GB+) and can be re-downloaded from Splice/Qobuz if lost
- **Do sync project files** — they are small (typically 50–500MB each) and irreplaceable

### Ableton "Collect All and Save"

This is the single most important backup command in Ableton:

1. Open your project
2. Go to **File → Collect All and Save** (or press Cmd+Shift+S on some systems)
3. Ableton copies every sample, preset, and file used by the project INTO the project folder
4. The project folder becomes fully self-contained — it can be opened on any computer with Ableton installed, even without the original sample library
5. **When to use:**
   - Before archiving any finished project
   - Before moving a project to a different computer
   - Before sharing a project with a collaborator
   - Monthly: run it on all active projects as part of your backup routine
6. **File size warning:** Collect All and Save can balloon project folders from 5MB to 500MB+ if the project uses many audio samples. This is expected and correct — the trade-off for portability is disk space.

### Version Control

Ableton does not have built-in version control. Manage versions manually:

- **Naming convention:** `TrackName_v1.als`, `TrackName_v2.als`, `TrackName_v3.als`
- **When to increment:** Before any major change (new arrangement, different drop approach, pre-mixdown). Save a new version BEFORE you make the change so you can always go back.
- **Minimum:** Save a new version at these milestones:
  1. `_v1` — Initial idea (drums + bass locked)
  2. `_v2` — Arrangement complete
  3. `_v3` — Mixdown ready
  4. `_v4` — Final master
- **Delete old versions** only after the track is released and you have confirmed the final version plays correctly. Until then, keep everything.
- **Alternative:** Save versions with dates: `TrackName_20260320.als`. Less ambiguous than version numbers when you come back months later.

### Monthly Backup Checklist

1. Run **Collect All and Save** on all active projects
2. Copy active project folders to external SSD (if not already there)
3. Sync project folders to cloud backup
4. Verify external SSD health: open 2–3 random projects from the external drive, confirm they load correctly
5. Delete any temporary bounced audio files (Ableton creates these during freeze/flatten operations)
6. Confirm Splice downloads are filing into the correct sample library folders (not accumulating in Downloads/)
